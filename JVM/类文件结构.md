## 虚拟机概述           
```
1，Java虚拟机不和包括java语言在内的其他语言进行绑定，只和class文件绑定，虚拟机不关心class文件来源于哪个语言。
2，任何一个class文件都有对应着唯一一个类或者接口，但是不是所有的类都对应有class文件，有些类可以通过加载器直接加载
```
## Class文件       
```
1，是一组以八字节为基础的二进制流
2，Class文件采用类似C语言的伪结构来存储，1，无符号数：基本数据类型，描述数组，索引引用，符合utf8编码的字符串值，数量值。
3，表，由多个无符号数或者其他表构成的复合数据结构，以_info结尾。
```
文件结构：
```
1,魔数和版本信息，开头四个字节，唯一作用是确认该文件是否是虚拟机接受的类型，后面两个字节是次版本号，7-8字节是主版本号，java的版本号从45开始
2，常量池的存储，（1 字面量，2 符号引用：类和接口的限定名，字段名称和描述，方法的名称和描述）
3，常量池容量计数器，contant_pool_count ，常量池开头的U2类型的无符号数，用来表示常量池一共有多少项常量
4，访问标志，用于识别一些类或者接口层次的访问信息，如是private或者public
5，类索引接口和接口索引集合：都是u2无符号数，接口索引集合是一组u2数据的集合，存放着implements实现的接口名称，其第一项是计数器，表示实现了多少个接口
6，字段表集合：描述接口或者类中声明的变量，包括类级变量以及实例变量。
7，方法表集合，描述访问标志。名称索引，描述符索引，属性表集合等
8，属性表集合
```
为什么java中类，接口，变量的名字要小于64k？
```
因为它们的名字都是有CONSTANT_UTF8_INFO类型来表示，这种类型用u2来存储字符串长度，由于两个字节最多表示65535个数，因此这些名字最多只能是64k
```
